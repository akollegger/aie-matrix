# Implementation Plan: H3GeoJSON Map Editor

**Branch**: `012-h3geojson-map-editor` | **Date**: 2026-04-28 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `specs/012-h3geojson-map-editor/spec.md`

## Summary

A standalone browser-based map editor (`tools/map-editor`) that authors `.map.gram` files directly in the H3GeoJSON format defined by RFC-0010. It forks the rendering approach of h3-viewer вЂ” H3 hex cells rendered on a MapLibre GL base map вЂ” and adds tile painting, polygon draw-and-fill, tile type definitions, item placement, portal authoring, and `.map.gram` export/import. No server component; all state is client-side and files are transferred via browser download/upload.

## Technical Context

**Language/Version**: TypeScript 5.7 (browser target, ESM)  
**Primary Dependencies**: React 18, Vite 6, MapLibre GL 5 (base map), h3-js 4 (cell math), `@relateby/pattern` (gram import parsing)  
**Storage**: Browser memory only (MVP); files exchanged via browser download/upload  
**Testing**: Vitest unit tests for io/export and io/import; browser smoke test via `pnpm dev`  
**Target Platform**: Modern desktop browser (Chrome / Firefox current release)  
**Project Type**: Browser web application (standalone SPA, no backend)  
**Performance Goals**: Polygon fill + tile render for 5,000-cell map (Moscone scale) completes in under 2 seconds; map panning/zooming at 60 fps  
**Constraints**: Client-side only; no API key required for base map tiles (OpenStreetMap via MapLibre GL); no Colyseus dependency  
**Scale/Scope**: Single-map authoring; up to ~10,000 tile instances per session

## Constitution Check

- **Proposal linkage**: RFC-0010 (`proposals/rfc/0010-h3geojson-map-editor.md`) directly specifies this feature. Scope matches.
- **Boundary-preserving**: New package `tools/map-editor` follows the `tools/` pattern alongside `tools/tmj-to-gram`. No changes to server, shared-types, or existing clients.
- **Shared interfaces**: The `.map.gram` output format is the cross-boundary contract; it must be consumed by the game engine without transformation (IC-001). Contract documented in `contracts/map-gram.md`.
- **Verifiable increments**: Eight user stories, each independently testable starting from Story 1 (view only). Smoke test: `pnpm dev` in `tools/map-editor`; export a `.map.gram` and load it in the game engine.
- **Documentation impact**: `proposals/rfc/0010-h3geojson-map-editor.md` status в†’ accepted; `docs/architecture.md` authoring workflow section updated; new `docs/guides/map-editor.md`.
