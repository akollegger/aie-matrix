# IC-004: World Event Envelope

**Status**: Accepted (Phase 2+)  
**Consumers**: Colyseus bridge (produces), ghost agents Listener/Social tier (consume), TCK Listener/Social suites  
**Supersedes**: [IC-008 spike-only synthetic event](../../008-a2a-ghost-house-spike/contracts/ic-008-spike-synthetic-world-event.md) — that was a draft placeholder; this is the production contract  
**Source of truth**: RFC-0007 §Colyseus Bridge

## Purpose

Define the JSON envelope that the Colyseus bridge produces when translating a Colyseus world event into an A2A event for delivery to a Listener or Social ghost agent. The envelope is carried as an A2A `data` part in the push notification task.

Wanderer agents (Phase 1) receive no events; this contract is not exercised until Phase 2.

## Envelope

```json
{
  "schema": "aie-matrix.world-event.v1",
  "eventId": "01JXXXXXXXXXXXXXXXXXXXXXXX",
  "ghostId": "01JXXXXXXXXXXXXXXXXXXXXXXX",
  "kind": "world.message.new",
  "payload": {},
  "sentAt": "2026-04-24T12:00:00.000Z"
}
```

### Field Rules

| Field | Type | Rule |
|-------|------|------|
| `schema` | string literal | Must be `"aie-matrix.world-event.v1"`; bump minor version on additive changes, major on breaking |
| `eventId` | ULID string | Unique per event; generated by bridge at translation time |
| `ghostId` | string | The ghost that is the target of this event |
| `kind` | WorldEventKind | One of the values below; scoped as `world.*` |
| `payload` | object | Event-specific JSON; typed per `kind` |
| `sentAt` | ISO-8601 UTC string | Bridge translation timestamp |

## Event Kinds and Payloads

### `world.message.new`

A partner or another ghost sent a message to this ghost.

```json
{
  "from": "01JXXXXXXXXXXXXXXXXXXXXXXX",
  "role": "partner",
  "priority": "PARTNER",
  "text": "Hello, ghost!"
}
```

| Payload field | Type | Notes |
|--------------|------|-------|
| `from` | string | Sender id (ghost id or partner id) |
| `role` | `"partner" \| "ghost"` | `"partner"` for human attendee; `"ghost"` for another ghost |
| `priority` | string | `"PARTNER"` for partner messages; priority level for ghost messages |
| `text` | string | Message content |

### `world.proximity.enter`

Another ghost entered an adjacent cell.

```json
{
  "ghostId": "01JXXXXXXXXXXXXXXXXXXXXXXX",
  "h3Index": "8f2830828052d25"
}
```

### `world.proximity.exit`

Another ghost left an adjacent cell.

```json
{
  "ghostId": "01JXXXXXXXXXXXXXXXXXXXXXXX",
  "h3Index": "8f2830828052d25"
}
```

### `world.quest.trigger`

A quest trigger fired at the ghost's current location.

```json
{
  "questId": "quest-001",
  "h3Index": "8f2830828052d25",
  "detail": {}
}
```

`detail` is quest-specific JSON; agents SHOULD handle unknown detail shapes gracefully.

### `world.session.start`

The ghost's Colyseus session began (ghost entered the world).

```json
{
  "h3Index": "8f2830828052d25"
}
```

### `world.session.end`

The ghost's Colyseus session ended (ghost left the world).

```json
{}
```

## A2A Delivery

The envelope is delivered as an A2A push notification. The bridge:

1. Wraps the envelope JSON in an A2A `Message` with a `data` part
2. Calls `setTaskPushNotificationConfig` on the agent session task *before* terminal state (required invariant from IC-002)
3. The agent task remains open (streaming) for event delivery

Agents MUST NOT assume events arrive in strict Colyseus order — network and bridge processing may reorder events. Agents SHOULD use `eventId` (ULID, time-sortable) for ordering if needed.

## Versioning

The `schema` field is the version discriminant. The Colyseus bridge stamps the version it produces; agents SHOULD reject envelopes with unknown `schema` values rather than silently ignoring them.

Breaking changes to the envelope shape (rename/remove fields) MUST increment the major version string (`v2`, `v3`, …) and require a coordinated bridge + agent release. Additive changes (new optional fields) MAY stay on the same version.

## TCK Validation (Phase 2)

The Listener TCK validates:
1. Agent receives an event with `schema === "aie-matrix.world-event.v1"`
2. `eventId` is a valid ULID
3. `ghostId` matches the target ghost
4. `kind` is one of the declared event kinds
5. `sentAt` is a parseable ISO-8601 UTC string
6. Agent does not emit `say` in response to received events (Listener non-speech property)
