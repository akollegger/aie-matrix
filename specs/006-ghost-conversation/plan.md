# Implementation Plan: Ghost Conversation Mechanics

**Branch**: `006-ghost-conversation` | **Date**: 2026-04-20 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/006-ghost-conversation/spec.md`

## Summary

Implement the ghost conversation model defined in RFC-0005 and ADR-0003: a `say`/`bye` MCP tool pair that gates ghost movement behind engine-enforced conversational state, a `server/conversation/` package with a pluggable JSONL store and HTTP read endpoints, and updates to `random-house`, `ghost-cli`, and the debug panel to exercise and observe the new behavior. A lightweight `inbox` polling tool bridges the gap until ghost houses have persistent Colyseus connections.

## Technical Context

**Language/Version**: TypeScript 5.7 / Node.js 24 (ESM, `"type": "module"`)  
**Primary Dependencies**: `effect` v3+, `@colyseus/core` 0.15.57, `@modelcontextprotocol/sdk` 1.29+, `zod` 3, `h3-js` (existing), `ulid` (new — message IDs)  
**Storage**: JSONL on disk (`{ghost_id}.jsonl` per thread); in-memory ghost state in `ConversationService`  
**Testing**: `pnpm test` (unit/TCK); smoke tests in `quickstart.md`  
**Target Platform**: Linux server (Node.js 24, monorepo combined process)  
**Project Type**: Monorepo — new server package + updates to existing ghost and client packages  
**Performance Goals**: `say` completes (persist + fan-out) within same MCP request lifecycle; HTTP store reads < 500ms for threads up to 100k records  
**Constraints**: Store interface uses `Promise<T>`, not Effect-ts — third-party contributor accessibility; JSONL store adequate up to ~100k messages  
**Scale/Scope**: PoC — conference-day volume per ghost; single combined server process

## Implementation Slices

Each slice is independently demonstrable and maps to a spec user story.

### Slice A — Conversational State + say/bye tools (US-1, US-2)

**Covers**: FR-003–FR-009, FR-017–FR-018

**What**: `server/conversation/` package with ConversationService and JSONL store; `say` and `bye` tools wired into `server/world-api/`; `go`/`traverse` enforcement; Colyseus schema addition.

**Verify**: `quickstart.md` smoke test 1 — say enters conversational mode, go is rejected with IN_CONVERSATION, bye restores normal, second go succeeds.

**Key decisions**:
- `ConversationService` is an Effect `Context.Tag` wired into the MCP request layer alongside `WorldBridgeService` and `RegistryStoreService`.
- Cluster computation: `h3.gridDisk(ghostCell, 1)` → `MatrixRoom.listOccupantsOnCell()` per cell → deduplicated ghost IDs excluding speaker.
- `say` fails atomically: if `store.append()` throws, state transition does not occur.
- `CONVERSATION_DATA_DIR` env var controls JSONL output directory (default: `./data/conversations/`).
- Ghost house JWT claims need `ghostHouseId` for HTTP auth in Slice C — verify `server/auth/src/jwt.ts` before starting Slice C; add if absent.

### Slice B — inbox + random-house conversation (US-4, research Decision 8)

**Covers**: FR-003–FR-009 (fan-out path), US-4 random-house

**What**: `inbox` MCP tool; random-house conversation loop.

**Verify**: `quickstart.md` smoke test 2 — multiple ghosts exchange messages; debug panel shows conversational state transitions.

**Key decisions**:
- `inbox` drains the in-memory notification queue atomically per call. Notifications lost on restart — acceptable for PoC.
- random-house behavior: 20% chance to `say` after `look` shows occupants; always respond to `inbox` notifications; 15% chance to `bye` per poll tick while conversational.
- Canned message pool: small array of strings selected at random.

### Slice C — HTTP store endpoints (US-3)

**Covers**: FR-013–FR-015

**What**: `ConversationRouter` mounted at `/threads`; ghost house API key auth; ULID cursor pagination.

**Verify**: `quickstart.md` smoke test 3 — list, paginate, fetch single message by ID.

**Key decisions**:
- Auth: check `ghostId` belongs to calling ghost house via `RegistryStoreService`.
- JSONL `list()`: reads and parses the full `.jsonl` file, filters by `after` ULID cursor, returns up to `limit`. Adequate for PoC volume.

### Slice D — ghost-cli conversation support (US-4 ghost-cli)

**Covers**: FR-003–FR-009 (client surface), US-4

**What**: `say`/`bye` as REPL commands and one-shot CLI subcommands; conversational mode indicator in status strip; message.new log entries; inbox polling in REPL loop.

**Verify**: `quickstart.md` smoke test 1 (full end-to-end with UI feedback).

**Key decisions**:
- `parseReplCommand` extension: `say <content>` (rest of line is content), `bye` (no args).
- Status strip: append `[conversational]` when in conversational mode.
- Inbox polling: every 3s in REPL loop; append returned notifications to log panel.

### Slice E — debug panel (US-4 debug)

**Covers**: SC-006, SC-007

**What**: `spectatorDebug.ts` shows `mode=conversational` per ghost; reacts to `ghostModes` Colyseus schema changes.

**Verify**: `quickstart.md` smoke test 4.

**Key decisions**:
- Subscribe to `room.state.ghostModes.onChange` (same pattern as `ghostTiles.onChange`).
